# 기획 화면 목업 (Planning Mockups)

home_agent의 UI를 React 구현 전 정적 HTML로 먼저 시각화한 화면들.
모든 화면은 동일한 사이드바 레이아웃 + 공유 CSS(`_shared/style.css`)를 사용한다.

## 화면 목록

| 파일 | 화면 | 담당 에이전트 / 영역 |
|---|---|---|
| `index.html` | 전체 화면 인덱스 (이 목업들의 진입점) | — |
| `dashboard.html` | 메인 대시보드 — 오늘 요약 (일정·TODO·가계부·알림) | 오케스트레이터 |
| `chat.html` | 오케스트레이터 챗 UI (스트리밍·툴 호출 트레이스) | 오케스트레이터 |
| `schedule.html` | 일정 + 비전/목표 캘린더 + 골 트리 | 일정 에이전트 |
| `ledger.html` | 가계부 — 거래 입력·카테고리·예산 | 가계부 에이전트 |
| `finance.html` | 자산·계좌·평가액 트래킹 | 재무 에이전트 |
| `ideas.html` | 아이디어/전략 노트, 태그·검색 | 아이디어 에이전트 |
| `files.html` | 문서/사진 정리 — 그리드, 메타데이터, 의미검색 | 파일 에이전트 |
| `todo.html` | 업무 TODO — 프로젝트·컨텍스트·우선순위 | TODO 에이전트 |
| `settings.html` | 설정 — 에이전트/MCP/모델/인증 | 시스템 |

## 사용법

각 HTML은 정적 파일이며 그대로 브라우저에서 열면 된다.
`planning/screens/index.html`을 열어 화면 간 이동.

## 디자인 원칙 (목업 기준)

- 라이트 톤, 모노스페이스 + sans 혼용, 좌측 고정 사이드바 + 우측 메인.
- Tailwind/shadcn 분위기를 흉내내되 의존성 없는 순수 CSS (CDN 미사용).
- 실제 데이터는 더미. 인터랙션 없음. 디자인·정보 구조 검증용.
- 폰트: 시스템 폰트 스택. 아이콘: 유니코드/이모지로 최소화.
